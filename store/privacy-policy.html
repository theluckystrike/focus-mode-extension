Privacy Policy

Focus Mode: Block Distractions & Stay Productive

Effective Date: February 12, 2026

The short version: Focus Mode Pro is a privacy-first extension. Your browsing history, personal information, and blocked site lists are never collected or transmitted. The extension makes limited network requests solely for license verification and anonymous analytics to the Zovo API. All focus session data and settings remain stored locally on your device.

1. Data Collection

Focus Mode Pro does not collect personal data. The extension does not collect, process, or transmit any browsing history, blocked site lists, or personally identifiable information beyond what is described in this policy.

All data generated by the extension — including your settings, focus session statistics, blocked site lists, and whitelist rules — is stored on your local device using Chrome's built-in storage APIs. This data never leaves your browser.

The extension does transmit limited data to the Zovo API for the purposes described in Section 2 (License Verification) and Section 3 (Analytics). This data does not include your browsing history, personal information, or blocked site lists.

2. License Verification Service

Focus Mode Pro connects to the Zovo API to verify Pro license keys. When you enter a license key or when the extension periodically re-validates your license, a request is made to the following endpoint:

  • Endpoint: https://xggdjlurppfcytxqoozs.supabase.co/functions/v1/verify-extension-license

Data transmitted during license verification:

  • Your license key
  • The extension ID

Data returned from the server:

  • License validity status and tier (e.g., Pro)
  • The email address associated with the license
  • Expiration date (if applicable)

How license data is stored:

  • Your license key is stored in chrome.storage.sync, which syncs across your signed-in Chrome devices
  • A license validation cache is stored in chrome.storage.local to reduce repeated network requests

License verification does not transmit your browsing history, blocked site lists, focus session statistics, or any other personal data. The free tier of the extension works fully offline without any license verification requests.

3. Analytics

Focus Mode Pro sends anonymous analytics events to the Zovo API to help us understand how the extension is used and to improve the product. These events are sent to the following endpoint:

  • Endpoint: https://xggdjlurppfcytxqoozs.supabase.co/functions/v1/track-event

Data included in each analytics event:

  • Event name (e.g., "session_started", "license_activated")
  • Extension ID
  • Session ID (a random identifier, not tied to your identity)
  • Timestamp
  • Extension version number

Data explicitly NOT included in analytics:

  • Browsing history or URLs visited
  • Personal information (name, email, location)
  • Blocked site lists or whitelist rules
  • Focus session content or scores
  • Any information that could identify you personally

If you are offline, analytics events are queued locally and sent when an internet connection becomes available. Analytics data is used solely for product improvement and is never sold to or shared with third parties.

4. Network Requests Summary

Focus Mode Pro makes network requests only to the Zovo API for the two purposes described above. The extension does not include:

  • No third-party analytics services (Google Analytics, Mixpanel, etc.)
  • No crash reporting or telemetry services
  • No remote configuration or feature flags
  • No advertisement networks
  • No connections to any servers other than the Zovo API

The core blocking functionality of the extension works entirely offline. An internet connection is only needed for license verification and analytics.

5. Permissions Explained

Focus Mode Pro requests only the permissions strictly necessary for its core functionality. Below is a full explanation of each permission and why it is required.

storage

Used to save your preferences, blocklist rules, whitelist rules, and focus session statistics locally on your device. No data is transmitted externally.

tabs

Required to detect when you navigate to a blocked site and redirect you to the blocked page. Tab information is processed locally and never transmitted externally.

webNavigation

Required to intercept navigation to blocked websites before the page loads, providing instant blocking without loading the distracting content first.

alarms

Used to manage focus session timers, break reminders, and scheduled focus mode activation. Runs entirely locally with no external communication.

notifications

Used to notify you when focus sessions complete, breaks end, or when it is time to take a break. All notifications are generated locally on your device.

host_permissions (<all_urls>)

Required to block distracting websites on any domain you configure in your blocklist. This permission only activates during focus mode and only applies to sites you have explicitly added to your blocklist or enabled via category toggles.

6. Data Storage

The extension uses two Chrome storage areas:

chrome.storage.local (device only):

  • Focus mode settings and preferences
  • Blocked site and whitelist rules
  • Category toggle states
  • Focus session statistics and streak data
  • Timer mode and duration preferences
  • Schedule configuration
  • License validation cache
  • Queued analytics events (sent when online)

chrome.storage.sync (synced across devices):

  • License key (so your Pro status follows you across devices)

You have full control over your data at all times. You can clear all stored data from the extension's options page, or by removing the extension from Chrome. The Zovo API does not persistently store any data beyond what is necessary for license management and aggregated analytics.

7. Third Parties

Focus Mode Pro communicates only with the Zovo API, which is hosted on Supabase infrastructure, for license verification and analytics as described in Sections 2 and 3. No data is shared with any other third parties.

The extension contains no third-party analytics libraries (such as Google Analytics), advertising frameworks, or external service integrations beyond the Zovo API. Analytics and license data are never sold to or shared with third parties.

8. Updates to This Policy

This privacy policy may be updated from time to time to reflect changes in the extension or to clarify existing practices. Any changes will be reflected on this page with an updated effective date.

We encourage you to review this policy periodically. Continued use of the extension after changes constitutes acceptance of the updated policy.
